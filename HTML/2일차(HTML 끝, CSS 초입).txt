// 웹 호스팅 : 깃허브 페이지

1. 호스트 : 인터넷에 연결된 컴퓨터 하나하나를 일컫음
2. 호스팅 : 웹 서버를 운영하기 위한 컴퓨터, 다시 말해 호스트를 빌려주는 사업을 호스팅이라고 한다( = 클라우드 )
3. 웹 호스팅 : 컴퓨터의 웹 서버와 같이 웹을 동작시키는데 필요한 소프트웨어를 설치해서 빌려주는 일

4. 깃허브 웹 호스팅 기능
: 'Settings'에서 'Pages'에 어떤 브랜치를 쓸 지 결정하고 내보낸다. 진행과정은 Action 탭에서 확인 가능

4-1) 깃허브 웹 호스팅 서비스를 활용해서 개발한 웹 페이지를 전세계 사람들과 공유할 수 있게 됐다.
( https://kdh829.github.io/test/ )

-> 수정하고 싶으면 html 소스코드를 수정해서 다시 업로드 한다. 그러면 파일 크기 등을 비교해서 수정된 부분만 다시금
업데이트 한다. 새로 업데이트된 파일은 새 버전으로써 업그레이드 되는 것임.

이 과정을 이론적으로 풀어보자면..

첫째, 나의 컴퓨터가 있고, 컴퓨터 내에 내가 만든 웹 페이지가 있다. 
둘째, 내 웹 페이지를 보고 싶어하는 방문자들이 있다.
셋째, 서비스를 대신해서 제공할 호스팅 업체인 깃허브가 있다.

내가 깃허브에 파일을 업로드하면 소스코드가 웹 호스팅으로 업로드 된다. 웹 호스팅 업체에서 웹 호스팅 환경에 설치된
웹 서버를 활성화 한다. 그리고 호스팅 컴퓨터에 도메인 이름을 부여한다. 다음으로 방문자가 도메인 주소로 접속하면
웹서버는 .html 문서를 읽는다. 그럼 .html 문서의 소스코드를 방문자에게 보내게 되고 방문자 컴퓨터가 이를 해석하여
웹 페이지가 시각적으로 보이게 되는 것이다.

// 웹 서버

웹브라우저와 마찬가지로 제품군이다. 그렇기 때문에 여러 제품들이 존재하는데, '아파치', 'iis', '엔진엑스' 등이 있다.
테스트용으로는 'web server for chrome' 이 무료이고 쓰기 좋다. 이 웹 서버가 대신 문서를 등록받아 외부에서 내 서버를
통해 html 문서로 접속할 수 있다. 외부에서 내 도메인 주소로 접속하면 내 웹 서버에서 문서를 코드로 보내어 외부 컴퓨터
에서 이를 해석하고 컴퓨터 화면에 보여주게 되는 것임.
( file://은 웹브라우저에서 직접접속, https://은 웹서버를 통해 접속해서 웹서버가 문서 코드를 읽고 돌려줬다는 뜻 )

HTTP:// 뒤에 있는 127.0.0.1은 Internet Protocol Address, IP 주소를 의미한다. 뒤에 : 띄우고 8887은 포트라는 개념이다.

같은 기기에 웹 브라우저와 웹 서버가 통신하며 웹 페이지를 주고받는다? 
현실에서는 지구 반대편의 HTML 문서가 내 컴퓨터에 있었던 것 마냥 접속이 가능하다. 
테스트 해볼 수 있는데 핸드폰의 탈을 쓴 컴퓨터인 스마트폰과 내 컴퓨터를 네트워크 연결하는 것임.
같은 네트워크 즉, 같은 와이파이로 접속해야 한다 -> 웹 서버의 설정을 바꿔야 함( 먼저 웹 서버를 끄고 바꿔줘야 함 )
-> 웹 서버를 활성화하고 만들어진 IP주소+포트 번호를 핸드폰에 입력하면 화면이 뜰 것임
