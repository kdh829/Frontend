///// CSS 기본 문법

1. Grid로 배치된 내용 분할하기
: 분할하기에 앞서서 CSS로 디자인을 할 때, 의미없는 태그 명을 붙여야 할 수도 있다. 이때 사용하는 태그가
무색무취와 같은 태그가 바로 흔히 보던 <div>이다. 참고로 div는 Division의 약자이다. 

<div>의 특징은 다음과 같다.
1. 블록 레벨 Element( 화면 전체를 사용하는 태그)이다.
2. 그렇기 때문에 사용하면 자동으로 줄바꿈이 일어난다.

<div>와 똑같은 목적이지만, 인라인 Element( 필요한 부분에만 사용하는 태그 )인 것이 <span>이다.
목적에 따라 사용하면 된다. 아직 감이 잘 오지 않기때문에 실습하면서 습득해야할 것.

만약, 두 개의 태그를 나란히 배치하고 싶으면 그것을 감싸는 부모 태그가 필요하다.
즉, 디자인 목적만으로 <div id="grid"> <div> </div> </div> 과정이 필요한 것이다.

여기까지는 프레임 색상 및 두께를 지정하는 방법이었다.
그럼 본격적으로 레이아웃 그리드를 생성하는 방법을 알아보자면..

앞서 display 속성으로 어떤 태그를 어떻게 표시할지 지정했는데, display 속성값으로 'grid'를 입력해줘야 한다.
그리드를 만들겠다!라고 알려주는 것. 그러고나서 'grid-template-columns: 150px 1fr'로 영역 분할을 해준다.
여기서 첫 칼럼에 대해 150px 만큼, 두번째 칼럼에 대해 1fr 만큼 공간을 주게된다.
1fr ?? 이게 뭘까? 예를 들어 1fr 1fr 2fr이라고 세 칼럼에 대해 할당했다고 치자.
그럼 총 4fr에 대해 첫 칼럼은 4분의 1영역만큼, 둘 째 칼럼도 4분의 1영역만큼, 셋 째 칼럼은 4분의 2 영역. 즉 절반 공간을
할당해준다는 의미인 것임. 즉, 150px만큼 주고 둘 째 칼럼에게 나머지 공간을 모두 주겠다는 뜻이 된다.

<grid> 태그는 들어가는 글자 수 만큼 크기가 자동으로 조절된다. 
다만, 최신 기술이기 때문에 사용 가능 브라우저를 확인해야 한다( https://caniuse.com/ )

<div>로 문단을 나란히 했다면, 여백 상 맞지 않을 수 있다. 이땐 F12 사용자 도구를 통해 검사하여 패딩이나 마진으로
해결하면 된다. 

하다가 주의할 것! 예시로 <ol> 태그는 자동 순서 번호 기입이라는 기능을 가진다. 근데 grid 속성 값을
ol에 줘버리면 나중에 본문에 들어가는 상황에 번거로운 일이 생길 수 있다. 이를 위해서 부모를 지정해줘야 한다.
( ex. ol { border-right: 5px; } -> #grid ol { border-right: 5px; } )이라고 ID가 그리드인 태그 아래에 존재하는 그리드
속성 태그임을 알려주는 것이다.

///// 미디어 쿼리 소개

1. 반응형 웹
: Responsive Web. 즉 반응형 웹으로 화면 크기 조절에 따라 웹의 각 요소들이 반응하여 화면 내용을 크기에 맞게
최적화해주는 기능을 한다.

2. 미디어 쿼리란? CSS를 통해 반응형 웹을 구현하는 것.
미디어 쿼리를 사용하는 방법은 @media 를 쓰는 것이다. (ex. @media(min-width: 800px) { div { display: none; }} = 속성) 
즉, 최소 800px부터는 화면 표시를 안 해주겠다는 뜻이다.

->> 이를 통해서 스마트폰의 가로/세로 모드에 따라, 또 여러 화면 특성에 따라서 특정 조건을 만족할 때만 쿼리 코드를 동작시키는
것이 가능하다 = '미디어 쿼리'
