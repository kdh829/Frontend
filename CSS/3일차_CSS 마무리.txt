///// CSS 기본

1. 미디어 쿼리 사용하기
1-1) 미디어 쿼리란 CSS를 통해 반응형 웹을 만드는 것이다. 미디어 쿼리를 사용하면 프레임 사이즈에 조건을 줘서
해당 프레임 사이즈에 맞게 화면 레이아웃을 수정할 수 있다.


2. 중복되는 CSS 소스 분리하여 사용하기
2-1) <Link> 
: 확장자가 '.css'인 파일을 별도 생성해서 반복되는 CSS 소스를 복붙한다. 이 CSS 파일을 HTML과 연결해야
하는데 이때 사용되는 태그가 <Link>이다. 이 태그를 사용해서 얻을 수 있는 효과는 무수히 많은 파일에
한꺼번에 적용해서 개발 시간을 단축시키는데 있다.

<Link>는 웹 브라우저에게 어떤 CSS 파일과 연결되어있는지 알려줄 수 있는 태그이다.
( ex. <link rel="stylesheet" href="style.css"/> )

( <link rel="stylesheet" href="style.css"/>에 대한 해석 )
1) rel="stylesheet" : 연결할 외부 리소스의 관계를 정의한다. 여기서는 이 리소스가 스타일시트임을 나타낸다.
스타일시트는 웹 페이지의 스타일( ex. 레이아웃, 색상, 폰트 등 )을 정의하는 CSS 파일을 의미한다.

2) href="style.css" : 연결할 외부 스타일시트 파일의 경로를 지정한다. 이 경우 'style.css'라는 파일을 가리키고 있습니다.
이 파일은 웹 페이지의 스타일 정보를 담고 있다.

왜 사용하는가? 

1. CSS 코드가 공통 파일이 아니라 모든 파일의 소스에 나열되어 있다면, 개별적으로 어떤 요소가 포함되어 있는지
알 수 없다. CSS 공통 파일을 만들면 이 파일 하나만 검토하면 되니 일관성 유지와 시간 절약 측면에서 좋다.

2. 재사용성이 높다. 사용자의 입장에서는 코딩의 원리가 뭔지 몰라도 Link로 웹 브라우저가 바라보게만 해줘도
형태 구현이 가능해진다. 웹이 1억 개 혹은 그 이상이더라도 이 파일 하나만 사용해줘도 동일한 효과를 낼 수 있다. 

만약, 어떤 웹에서 어떤 CSS 파일을 다운받아 사용했는지 보고싶다면 사용자 도구->네트워크로 보면 알 수 있다.

+)
여러 개의 디자인 파일을 만들어두고 다운받는 것과 직접 소스 내 하드코딩을 해주는 방식 중 어떤 것이 효율적이라고
한다면 하드코딩을 해주는 것이 더 효과적일 수 있다. 그렇지만, '캐싱' 기법때문에 CSS 파일을 별도 생성 후에 적용하는
것이 더 효율적이다라는 말이 있다.

'캐싱'은 저장하다라는 의미이다. 파일이 수정되기 전까지는 컴퓨터에 쭉 저장 중이다가 필요 시에 가져와 쓸 수 있다.
이는 컴퓨터 트래픽을 줄이고, 속도를 높일 수 있으며, 사업자들은 비용을 절약할 수 있다는 장점이 있다. 


3) <div>와 <span>의 차이

3-1) <div> 태그는 화면 전체를 사용하기 때문에 줄바꿈이 이뤄진다. 크기에 따라 영역이 정해지는 방법을 써서
CSS 적용 범위를 좁혀주고 싶을 때 쓰는 문법이 <span>이다. <span>도 <div>와 마찬가지로 별다른 의미가 없는
태그이다.
