///// 해석

1. 카페24 웹호스팅 서버에서 우리의 웹 프로젝트를 배포할 수 있다.

1-1) 순서
1-1-1) 카페24에서 웹 호스팅 서버 이용권을 구매한다.
1-1-2) 카페24에서 구매한 DB 계정 정보로 DAO 클래스를 수정해준다.
1-1-3) 우리가 설정한 JAVA의 JDK 버전이 카페24 서버의 JDK 버전보다 높으면, '설정'에서 'Project facets'에서 JAVA의 버전을 낮춰서 적용해준다.
1-1-4) 마지막으로 만든 모든 파일을 웹 서버에 올리는 단계이다. '설정'에서 '리소스' 항목의 ROOT를 복사해서 들어간다.
1-1-5) 파일들을 웹 서버에 업로드하여 정상 작동시킬 수 있다. 웹 서버에 접속할 수 있는 웹 서버 도구가 필요하다. R드라이브가 필요.
1-1-6) FTP에서 사이트 정보에 호스트, 아이디와 비밀번호를 입력한다.
1-1-7) www 파일 내에 BBS라는 폴더를 또 만들어주고 모든 파일을 업로드 해준다( WEB, META-INF 제외 )
1-1-8) class와 lib 폴더에 나눠서 jar 등을 추가해준다.

( COMPILE 오류 발생 시 )
1-11-9) 컴파일 오류 건은 puTTY 프로그램에서 호스트 명 SSH 로 접속하여 비번을 입력 후 접속한다. 이후에 톰캣을 강제종료(Shutdown)하고
다시 재시작(Startup)하여 접속할 수 있도록 한다.
